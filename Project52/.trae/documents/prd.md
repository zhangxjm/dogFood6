## 1. 产品概述

航天载荷试验数据管理系统，面向航天实验室，解决试验数据非标准化、采集流程不规范、报告生成效率低等核心问题。系统对接实验室设备与SQLite数据库，实现试验数据全生命周期管理，从采集、解析、标准化到报告自动生成，全程可追溯、可管控。

- 目标用户：航天实验室工程师、试验管理人员、质量审核人员
- 核心价值：消除非标数据孤岛，规范试验流程，将报告生成时间从数小时缩短至秒级

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 试验工程师 | 系统自动分配 | 创建试验、采集数据、查看报告 |
| 试验主管 | 系统自动分配 | 管理试验流程、审核数据、导出报告 |
| 系统管理员 | 系统自动分配 | 设备管理、用户管理、系统配置 |

### 2.2 功能模块

1. **仪表盘**: 系统概览、试验状态统计、近期活动、关键指标
2. **试验管理**: 试验创建/编辑、流程管控、状态追踪、里程碑管理
3. **数据采集**: 设备数据接入、实时采集监控、批量数据导入、数据校验
4. **数据解析**: 非标数据解析引擎、格式转换、数据标准化、解析规则配置
5. **报告中心**: 报告模板管理、自动生成报告、报告预览/导出、历史报告查询
6. **设备管理**: 设备注册/注销、设备状态监控、校准记录、通信协议配置

## 3. 核心流程

### 试验数据全生命周期流程

用户创建试验 → 配置关联设备 → 启动数据采集 → 实时数据流入系统 → 非标数据自动解析 → 格式标准化处理 → 数据校验与存储 → 触发报告自动生成 → 报告预览与导出

```mermaid
flowchart TD
    "A[创建试验]" --> "B[关联设备]"
    "B" --> "C[启动数据采集]"
    "C" --> "D[实时数据流入]"
    "D" --> "E[非标数据解析]"
    "E" --> "F[格式标准化]"
    "F" --> "G[数据校验存储]"
    "G" --> "H[报告自动生成"
    "H" --> "I[预览与导出]"
    "C" --> "J[异常监控]"
    "J" --> "K[异常处理]"
    "K" --> "C"
    "E" --> "L[解析失败]"
    "L" --> "M[规则调整]"
    "M" --> "E"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深空蓝 (#0A1628) + 星际蓝 (#1B3A5C)，强调科技感与航天主题
- 辅助色：信号绿 (#00E676) 表示正常/在线，警告橙 (#FF9100) 表示异常，危险红 (#FF1744) 表示故障
- 按钮风格：圆角矩形（6px），主按钮实心填充，次按钮描边
- 字体：标题使用 Orbitron（科技感显示字体），正文使用 Noto Sans SC（中文优化）
- 布局风格：左侧固定导航栏 + 右侧内容区，卡片式信息展示
- 图标风格：线性图标（Lucide Icons），统一2px描边
- 背景：深色主题为主，渐变叠加，微粒子效果点缀
